Launch Standard Documentation

The operating manual for business-complete builds.

Your Deputy is designed to produce more than a website. Every validated build is checked for business identity, funnel pages, products, CRM, RevOps, contracts, invoices, payments, onboarding, support, PostHog-ready analytics, AI agents, MCP tools, open-weight model routing, optional media assets, workflows, security, resumable repair evidence, and self-hostable infrastructure.

Build Contract

Each project is validated against the autonomous business-unit standard before it can report BUILD STATUS: PASS.

Required Modules

Lead capture, info products, quotes, proposals, contracts, invoices, payment abstraction, CRM, onboarding, support, analytics, workflow engine, and MCP-compatible tools are mandatory for full launches.

Central Config

Generated builds use a single master configuration contract for business identity, provider choices, secrets placeholders, feature flags, AI, billing, email, storage, analytics, and workflows.

Provider Boundaries

External services are optional adapters. Core business logic must route through internal interfaces for payments, email, storage, analytics, AI, documents, signatures, and notifications.

Validation Gates

  • Route and live-content checks
  • Schema and workflow checks
  • MCP registry and permissions checks
  • Revenue, RevOps, CRM, support, and analytics checks

Deployment Model

Every full build targets Docker-first, self-hostable infrastructure with explicit degraded-but-working scaffolds when credentials are unavailable.

Repair & Resume

Build failures are treated as structured inputs: detectors classify repo, route, deployment, quality, and live-content issues; remediators repair what they can; checkpoints let future runs continue from the last verified stage.

Model Routing

Generated evidence includes an open-weight model catalog and routing notes for GPT-OSS, Gemma, Qwen, DeepSeek, Phi, Devstral, Magistral, and hosted fallbacks where configured.

Media & Analytics

Optional video-use and VibeVoice-compatible planning create niche video and voice asset scaffolds, while PostHog-ready events connect acquisition, product usage, revenue, and customer lifecycle analytics.

Accountability rule: Vercel green is not enough. A build is ready only when code, routes, live content, quality contract, required modules, model/tool evidence, analytics, repair telemetry, and generated business artifacts all verify together.
