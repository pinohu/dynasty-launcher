Production capability

Business Factory Console

This page reflects what is actually deployed in production today. The backend generates a complete launch manifest, validates production readiness, produces a revenue and funnel plan, and can enqueue a durable business_factory.launch job.

Delivered nowIdea scoring, offer creation, generated app manifest, funnel, Stripe catalog, automations plan, deployment contract, agent roles, and queue handoff.
Production URLThe APIs run on this public Vercel domain behind admin authentication.
Operational boundaryQueued jobs are durable. A worker or operator still has to claim and execute the job phases.
No hidden promiseVendor actions depend on configured credentials, tier allowlists, provider API health, and the worker loop.
Live backend surfaces
  • POST /api/business-factory
    Generate and optionally enqueue a business launch manifest.
  • POST /api/jobs/enqueue
    Create durable jobs with idempotency keys.
  • POST /api/jobs/claim
    Lease the next due job for a worker.
  • POST /api/jobs/complete
    Complete, fail, or retry a claimed job.
  • Admin required
    Use your production admin key. The key stays in this browser request and is never rendered by the page.

Production result

What buyers can assess

Exact generated outputs: offer ladder, app routes, data model, funnel sections, email sequence, Stripe catalog, automation workflows, deployment requirements, and agent roles.

What operators can use

A production admin can generate manifests, enqueue launch jobs, list queue state, and wire a worker to claim and complete jobs.

What is not implied

Public visitors cannot trigger vendor spend. Queued jobs do not execute themselves unless a worker loop is running with the required credentials.
