# `doc/` — pointer to canonical documentation

There is no separate `doc/*.md` tree in this repository. All Markdown
documentation lives under **`docs/`** (architecture, operations, strategy,
catalog, UX bible, JTBD, and CSV traces).

- **Index:** [`docs/architecture/README.md`](../docs/architecture/README.md)
- **Operations:** [`docs/operations/README.md`](../docs/operations/README.md)

Deliverable **playbooks** shipped as static HTML live under
[`deliverables/`](../deliverables/) (for example
[`deliverables/operations/operations-playbook.html`](../deliverables/operations/operations-playbook.html)).

The **`automations/`** and **`product/`** folders are the sister packages to
`docs/` — machine-readable catalog plus product model; see
[`automations/README.md`](../automations/README.md) and
[`product/README.md`](../product/README.md).

**New here?** Read [`START-HERE.md`](./START-HERE.md) first — it ties public pages, checkout, and `api/provision.js` together.
