What’s included today

This page exists for one reason: clarity before purchase. It shows what is live now, what is automation-ready, and what is key-gated so you can decide with confidence and zero surprises.

Last reviewed: 2026-04-12. Foundation focuses on strategy + product + deployment, while higher tiers unlock broader autonomous runtime capacity. Integration modules are implemented in the orchestrator and run automatically; execution still depends on credentials, provider API health, and approvals. The builder runs multi-pass post-deploy checks (repo paths, Vercel READY, strict live URL + routes) and an artifact quality gate on key docs (placeholders, template-leak patterns, niche grounding) before calling a deploy “ready.” The U.S. government contracting planning pack is opt-in in the builder and is informational only (verify on SAM, USASpending, SBA, Census NAICS).

Technical verification endpoint: /api/provision?action=inventory.

Status legend

  • Live — In active use for paying builds, subject to tier, keys, and API health.
  • Partial — Works in many cases but has known provider/key dependencies or reduced fallback behavior.
  • Codegen — Client-side generation + repo files; no external API required.
  • Automation-ready — Implemented server-side and auto-runs when credentials/provider APIs are available.
