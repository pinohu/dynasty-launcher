What’s included today

This page exists for one reason: accountability before purchase. It shows what is live now, what is wired server-side when keys allow, and what stays in your repo only — so you are not sold a vague “AI did it” story.

Last reviewed: 2026-04-12. Five tiers are available today:
Free Scoring ($0) — viability scorecard only.
Scoring Pro ($19/mo) — unlimited scoring with priority models.
Blueprint ($297) — deep diagnostic + execution plan.
Strategy Pack ($697) — all 90+ documents, downloadable .zip, no code or deployment. $697 credited toward Foundation within 30 days.
Foundation ($1,997) — everything in Strategy Pack plus production code, deployment, and Day-1 Success Kit.

Items marked “Planned” are in active development for future operational tiers. Credential boundary: launcher-held keys (e.g. DYNASTY_TOOL_CONFIG) are for generating deliverables and orchestrating setup; sustained operation of deployed apps is intended to use customer-owned vendor credentials in the shipped project (.env.example, MANUAL-ACTIONS, their Vercel env) — not an ongoing dependency on the launcher’s shared API pool. Customer Vercel projects are not auto-filled with real Clerk/Stripe/DB secrets from the launcher (placeholders + dashboard handoff). Modules are implemented in the orchestrator and execute as software workflows; success still depends on your approvals, API health, and tier limits. The builder runs multi-pass post-deploy checks (repo paths, Vercel READY, strict live URL + routes) and an artifact quality gate on key docs (placeholders, template-leak patterns, niche grounding) before calling a deploy “ready.” The U.S. government contracting planning pack is opt-in in the builder and is informational only (verify on SAM, USASpending, SBA, Census NAICS).

Technical verification endpoint: /api/provision?action=inventory.

Status legend

  • Live — In active use for paying builds, subject to tier, keys, and API health.
  • Partial — Works in many cases but has known provider/key dependencies or reduced fallback behavior.
  • Codegen — Client-side generation + repo files; no external API required.
  • Planned — Designed and specified; not yet available to customers. Will activate in future tiers when implementation is complete.
