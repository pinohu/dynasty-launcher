# Tenant template

Copy this folder to `tenants/<slug>/` to create a new tenant (the `init` CLI command does this for you).

## Files

| File | Purpose |
|---|---|
| `tenant.yaml` | Business config + infra references. Conforms to `schemas/tenant.schema.json`. |
| `.secrets/env.local.example` | Rename to `env.local` and fill in per-tenant secrets (gitignored). |
| `selected-automations.yaml` | Generated by `interview` — ordered list of picked automations. |
| `plan.json` | Generated by `plan` — deployment plan with waves + credential gap. |
| `deployed-automations.yaml` | Generated by `deploy` — what's actually running. |
| `status.json` | Generated by `verify` — latest health check. |
| `history/` | JSONL audit log of every deploy/rollback action. |
| `MANUAL-ACTIONS.md` | Generated — steps the tenant must perform manually (vendor account setup, DNS, etc.). |

## Lifecycle

```
init  →  interview  →  plan  →  deploy  →  verify
                                         ↑
                                         └──  rollback  →  upgrade ...
```
